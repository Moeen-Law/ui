# Moeen UI

UI for the Moeen law platform, built with React, TypeScript, Vite, Tailwind CSS, and shadcn-style components.

## Overview

Moeen is a legal AI assistant interface for Arabic-first legal workflows. The frontend provides authentication, chat, landing pages, internationalization, RTL-aware UI, and deployment-ready production assets.

## Tech Stack

- React 19 and TypeScript
- Vite 7
- Tailwind CSS 4 with typography support
- React Router 7
- TanStack Query
- Zustand
- React Hook Form and Zod
- i18next and react-i18next
- Framer Motion
- Axios and SSE streaming via `@microsoft/fetch-event-source`

## Project Structure

```text
src/
  assets/             Static assets imported by the app
  components/ui/      Shared UI primitives
  features/
    auth/             Authentication pages, forms, schemas, services, store
    chat/             Chat UI, streaming hooks, services, history, sidebar
    landing/          Public landing page
  lib/                Library setup such as i18n and utilities
  locales/            Arabic and English translation files
  routes/             Route configuration
  shared/             Shared API, hooks, pages, types, and constants
  test/               Local test/demo components
```

Other useful folders:

- `public/`: static files served by Vite, including brand assets and favicon.
- `dist/`: generated production build output.
- `desired-ui/`: reference UI assets and prototypes.
- `sse-guide/`: streaming-related support notes.

## Getting Started

### Prerequisites

- Node.js 18 or newer
- npm

### Install

```bash
npm install
```

### Environment

Create `.env` from `.env.example` and configure the API gateway:

```env
VITE_BASE_URL=https://gateway.moeenlaw.com
```

### Run Locally

```bash
npm run dev
```

### Production Build

```bash
npm run build
npm run preview
```

## Available Scripts

- `npm run dev`: start the Vite development server.
- `npm run build`: run TypeScript build checks and generate `dist/`.
- `npm run lint`: run ESLint for the project.
- `npm run preview`: serve the production build locally.
- `npm run docker:build`: build the Docker image as `registry.moeenlaw.com/ui:latest`.
- `npm run docker:push`: push the Docker image to the configured registry.

## Key Features

- JWT-based authentication flow with account verification and password reset screens.
- AI chat with SSE streaming, stop-generation support, Markdown rendering, and optimized scrolling behavior.
- Arabic and English localization with RTL-aware rendering.
- Responsive desktop and mobile layouts.
- Docker and Nginx deployment configuration.
- Woodpecker CI configuration.

## Development Notes

- Keep feature-specific code inside `src/features/<feature>/`.
- Put cross-feature utilities and types in `src/shared/`.
- Use PascalCase for React components and camelCase for hooks, helpers, and variables.
- Run `npm run lint` and `npm run build` before opening a pull request.
- Do not edit `dist/` manually; it is generated by the build.

## Docker

```bash
docker build -t moeen-ui .
docker run -p 80:80 moeen-ui
```

The npm scripts target the production registry image:

```bash
npm run docker:build
npm run docker:push
```

## Configuration and Security

- Do not commit secrets, local tokens, or private certificates.
- Keep environment-specific values in `.env`.
- Review changes to `Dockerfile`, `nginx.conf`, and `.woodpecker.yaml` carefully because they affect deployment.
